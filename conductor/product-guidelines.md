# Product Guidelines - RCN Server

These guidelines define the principles and standards for the development and documentation of the RCN Server.

## 1. Tone and Voice
- **Technical and Direct:** Documentation and communications should be clear, efficient, and use precise technical terminology. Avoid unnecessary fluff.

## 2. Design Philosophy
- **Flexibility First:** The system must be easy to extend. Prioritize making it simple for users to add custom scripts, scheduled functions, and new reconnaissance logic.

## 3. Development Principles
- **Modular Design:** Keep components isolated. This supports rapid iteration, easier testing, and the ability to replace or upgrade parts of the system without impacting the whole during the alpha phase.
